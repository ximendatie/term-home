# OpenSpec Workspace（Spec-Coding 模式）

本目录用于以 **spec-coding** 方式推进 `term-home`：

1. 先明确目标与范围（spec）。
2. 将需求拆解为可验证任务（tasks）。
3. 用最小实现落地核心功能（code）。
4. 根据运行结果持续回写规范（feedback loop）。

## 目录约定

- `specs/<spec-name>/spec.md`：需求、约束、验收标准。
- `specs/<spec-name>/plan.md`：技术方案与阶段计划。
- `specs/<spec-name>/tasks.md`：执行任务清单（可勾选）。

## 当前激活规格

- `core-mvp`：首个可运行版本（核心功能优先）。
